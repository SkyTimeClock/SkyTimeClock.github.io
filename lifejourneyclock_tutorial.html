Chapter 01

網頁布局與 CSS 架構

了解如何用 CSS 構建一個兼顧視覺質感與響應式能力的頁面骨架， 包含固定背景層、彈性主體排版，以及細節字體策略。

1.1固定背景層技巧

背景 Canvas 使用 position: fixed + inset: 0，讓它永遠固定在視口中，不隨頁面滾動而移動，且不佔據文件流空間。

背景 Canvas 定位CSS
#bgCanvas {
  position: fixed;
  inset: 0;              /* 等同於 top:0; right:0; bottom:0; left:0 */
  width: 100vw;
  height: 100vh;
  z-index: 0;
  pointer-events: none; /* ⬅ 關鍵！讓滑鼠事件穿透到下方元素 */
}
⚠️ 常見陷阱

忘記加 pointer-events: none 的話，背景 canvas 會攔截所有點擊事件，導致頁面上的按鈕、連結全部失效！

1.2Flexbox 垂直居中佈局

主體使用 body 作為全高 Flex 容器，搭配 flex-direction: column，讓所有子元素自動垂直排列於中央。這是最簡潔的垂直居中方案之一。

body 全頁 Flex 容器CSS
html, body {
  width: 100%;
  min-height: 100vh;     /* 最少填滿視口，超出可捲動 */
  display: flex;
  flex-direction: column;
  align-items: center;   /* 水平居中 */
  justify-content: center; /* 垂直居中 */
  overflow-x: hidden;   /* 防止橫向捲軸 */
}

.scene {
  position: relative;
  z-index: 1;           /* 確保在 bgCanvas 之上 */
}
💡 min-height vs height

用 min-height: 100vh 而非 height: 100vh，是因為如果頁面內容超過視口高度，height: 100vh 會裁切內容；min-height 則允許頁面自然延伸。

Chapter 02

背景填充圖案 設計原理

背景並非圖片，而是用程式即時繪製的大量自然主題圖案。 學習如何用「網格鋪排 + 隨機偏移」製造既有規律又不失自然感的紋理背景。

2.1核心策略：網格 + 偏移

直覺上，若要把圖案隨機分佈於整個畫面，可能想到「隨機 x、隨機 y」——但這樣容易產生圖案扎堆或大面積空白。更優雅的方案是先建立網格，再在每個格子內加入隨機偏移：

互動示範 — 網格 vs 純隨機 vs 網格偏移
點擊按鈕觀察不同分佈方式的視覺差異
網格偏移核心邏輯JS
const TILE_W = 52, TILE_H = 52; // 格子大小

const cols = Math.ceil(W / TILE_W) + 2; // 多出 2 列避免邊界空白
const rows = Math.ceil(H / TILE_H) + 2;

for (let row = -1; row < rows; row++) {
  for (let col = -1; col < cols; col++) {
    const rnd = seededRand(col * 997 + row * 4001 + 7);

    // 格子左上角 + 在格子內隨機偏移 (0.55 的格子寬度範圍)
    const tx = col * TILE_W + rnd() * TILE_W * 0.55 + TILE_W * 0.2;
    const ty = row * TILE_H + rnd() * TILE_H * 0.55 + TILE_H * 0.2;
  }
}

2.2Seeded Random — 可重現的隨機

問題：如果每次 resize 都用 Math.random()，視窗大小改變後所有圖案位置都會變化，讓使用者感覺畫面「閃動」。

解法：用「種子偽隨機」（Seeded Pseudo-Random），輸入同樣的種子永遠產生同樣的亂數序列，確保 resize 後圖案位置不變。

線性同餘產生器（LCG）JS
// 頁面載入時隨機一次，刷新後才變化
const PAGE_SEED = Math.floor(Math.random() * 0xffffff);

function seededRand(seed) {
  let s = (seed ^ PAGE_SEED) >>> 0; // XOR 混入頁面種子
  return function() {
    // 線性同餘演算法 (LCG) — 快速、均勻分佈
    s = (s * 1664525 + 1013904223) & 0xffffffff;
    return (s >>> 0) / 0xffffffff; // 回傳 [0, 1) 範圍
  };
}

// 使用方式：每格有自己獨立的 rnd 函數
const rnd = seededRand(col * 997 + row * 4001);
rnd() // 第一次呼叫 → 隨機值（決定 x 偏移）
rnd() // 第二次呼叫 → 不同隨機值（決定 y 偏移）
rnd() // 第三次呼叫 → 決定圖案類型
1664525乘法常數（來自 Knuth 的推薦值）
1013904223加法常數（保證週期性）
& 0xffffffff限制在 32-bit 整數範圍
>>> 0轉為無符號整數

2.3深度漸層遮罩

背景圖案繪製完後，疊上兩道半透明漸層遮罩，製造出「下方和右側越來越暗」的景深效果，讓主體時鐘更突出：

雙向遮罩層JS
// 垂直遮罩：上亮下暗
const maskV = bctx.createLinearGradient(0, LH*0.35, 0, LH);
maskV.addColorStop(0,   'rgba(80,130,40,0)');   // 從中段開始
maskV.addColorStop(0.5, 'rgba(80,120,38,0.45)');
maskV.addColorStop(1,   'rgba(60,95,30,0.85)');  // 底部 85% 遮蓋力
bctx.fillStyle = maskV;
bctx.fillRect(0, 0, LW, LH);

// 水平遮罩：右側加深
const maskH = bctx.createLinearGradient(LW*0.45, 0, LW, 0);
maskH.addColorStop(0, 'rgba(70,110,35,0)');
maskH.addColorStop(1, 'rgba(55,88,28,0.40)');
bctx.fillStyle = maskH;
bctx.fillRect(0, 0, LW, LH);
🎨 設計思維

遮罩顏色選用與背景漸層相同色系（深綠調），而非純黑色。這樣遮蓋後圖案「融入」背景，而不是被黑色壓死，視覺上更自然。

2.4圖案函數設計規範

所有圖案繪製函數遵循統一的簽名規格，方便在陣列中統一呼叫：

統一函數簽名JS
// 所有圖案函數都接受相同參數
function drawLeaf(c, x, y, size, angle, col, veinCol) { ... }
function drawBird(c, x, y, size, angle, col, veinCol) { ... }
function drawFlower(c, x, y, size, angle, col, veinCol) { ... }

// 圖案配置陣列
const MOTIFS = [
  { fn: drawLeaf,   sMin: 36, sMax: 55 },
  { fn: drawBird,   sMin: 26, sMax: 40 },
  ...
];

// 統一呼叫：不需要 if/switch 判斷類型
const m = MOTIFS[Math.floor(rnd() * MOTIFS.length)];
m.fn(bctx, tx, ty, size, angle, colStr, veinCol);
✨ 設計模式：策略模式（Strategy Pattern）

將所有圖案函數放入陣列，透過統一介面呼叫，稱為「策略模式」。新增圖案只需要加入陣列，不需修改主程式邏輯。這是讓程式具備可擴展性的關鍵技巧。

Chapter 03

座標系統 與變換

Canvas 2D 的座標系統是所有繪圖操作的基礎。 理解原點、軸方向、DPR 縮放、以及 translate/rotate/scale 的組合順序， 是繪製精確動畫的前提。

3.1Canvas 座標系統基礎

Canvas 的座標原點 (0, 0) 在左上角，X 軸向右為正，Y 軸向下為正（與數學直覺相反）。角度也是順時針為正。

Canvas 座標系統示意圖
注意：Y 軸向下為正，角度以 12 點鐘方向為 -π/2
⚠️ 時鐘角度的關鍵換算

時鐘 12 點在頂部，但 Canvas 預設 0 弧度指向 右方（3 點）。因此所有時鐘角度都需要減去 π/2：

const hA = hFrac * Math.PI * 2 - Math.PI / 2;
//                                   ↑ 旋轉 -90° 讓 0 對應 12 點方向

3.2DPR — Retina 高解析度處理

在 Retina / HiDPI 螢幕上，一個 CSS 像素對應多個物理像素（DPR 通常為 2 或 3）。若不處理，Canvas 圖案會模糊。

DPR 高清處理公式JS
const dpr  = window.devicePixelRatio || 1;
const SIZE = 640; // 邏輯尺寸（CSS px）

// 物理像素：Retina 上為 640 × 2 = 1280
canvas.width  = SIZE * dpr;
canvas.height = SIZE * dpr;

// CSS 尺寸仍維持邏輯大小，版面不跑位
canvas.style.width  = SIZE + 'px';
canvas.style.height = SIZE + 'px';

// 縮放繪圖上下文，使後續繪圖座標仍以 SIZE 為單位
ctx.setTransform(dpr, 0, 0, dpr, 0, 0);
canvas.width物理像素數量（影響畫質）
canvas.style.widthCSS 顯示尺寸（影響佈局）
ctx.setTransform縮放繪圖座標對應物理像素

3.3save() / restore() — 上下文狀態棧

ctx.save() 和 ctx.restore() 是 Canvas 繪圖中最重要的一對操作。它們保存和恢復整個繪圖上下文的狀態（包含：當前變換矩陣、strokeStyle、fillStyle、lineWidth 等所有屬性）。

互動示範 — save/restore 效果
沒有 save/restore：變換會累積影響後續繪圖
正確的使用模式JS
function drawLeaf(c, x, y, size, angle, col) {
  c.save();              // 📌 1. 保存當前狀態

  c.translate(x, y);    // 移動原點到繪製位置
  c.rotate(angle);      // 旋轉角度
  c.strokeStyle = col;  // 修改顏色屬性

  // ... 繪圖 ...

  c.restore();           // 📌 2. 恢復到 save() 之前的狀態
  // ↑ 無論在函數內做了什麼，都不會影響外部的繪圖上下文
}

3.4變換順序的重要性

在 Canvas 中，translate、rotate、scale 的順序非常重要，順序不同結果完全不同。規則：後寫的變換先執行（因為矩陣乘法的特性）。

互動示範 — 變換順序差異
選擇順序觀察差異
角色放置在軌道上的方法JS
const placeChar = (drawFn, orbitR, angle, frame, scale) => {
  // 計算角色在軌道上的座標
  const x = CX + Math.cos(angle) * orbitR;
  const y = CY + Math.sin(angle) * orbitR;

  ctx.save();
  ctx.translate(x, y);          // 1. 移動到軌道位置
  ctx.rotate(angle + Math.PI/2);  // 2. 旋轉使角色面向行進方向
  ctx.scale(scale, scale);       // 3. 縮放
  drawFn(frame);                  // 角色繪製函數以 (0,0) 為中心
  ctx.restore();
};
Chapter 04

JS 動畫架構 與事件

掌握 requestAnimationFrame 驅動的主迴圈架構、 時間差計算、以及讓動畫在不同裝置上保持一致速度的技巧。

4.1requestAnimationFrame 主迴圈

requestAnimationFrame（簡稱 RAF）告訴瀏覽器：「下次螢幕刷新前，請呼叫這個函數」。瀏覽器會自動配合螢幕更新頻率（60Hz → 每 16.67ms 一次），比 setInterval 更省電、更流暢。

RAF 動畫迴圈架構JS
let lastTs = null;

function drawFrame(ts) {  // ts = 時間戳記（毫秒）
  // ── 計算時間差 ──
  if (!lastTs) lastTs = ts;
  const dt = Math.min((ts - lastTs) / 1000, 0.05);
  //                                          ↑ 上限 50ms：切換分頁回來不會暴衝
  lastTs = ts;

  // ── 清除上一幀 ──
  ctx.clearRect(0, 0, SIZE, SIZE);

  // ── 繪圖（順序很重要，後繪的在上層）──
  drawFace();
  drawOrbitRing(...);
  drawHand(...);
  drawCenter();
  placeChar(...);

  // ── 驅動下一幀 ──
  requestAnimationFrame(drawFrame);
}

requestAnimationFrame(drawFrame); // 啟動迴圈
⚠️ Math.min(dt, 0.05) 的必要性

當使用者切換分頁後回來，瀏覽器暫停了 RAF，此時 ts 差值可能超過 1 秒。若不限制，dt 很大，走路動畫會瞬間跳幾十幀。Math.min(..., 0.05) 確保每幀最多前進 50ms，動畫不會「暴衝」。

4.2時間驅動的走路動畫

角色走路速度以「每秒換幾幀」為單位，乘以時間差 dt 累積，突破整數時換幀：

時間差驅動幀動畫JS
let walkTimers = { man: 0, woman: 0, child: 0 };
let walkFrames = { man: 0, woman: 0, child: 0 };

// 走路速度（每秒換幾幀）
const WALK_SPD = { man: 0.6, woman: 2.6, child: 8.0 };
//                 ↑ 男士慢  ↑ 女士中速    ↑ 小孩跑最快（秒針）

// 在每一幀 drawFrame 內執行：
walkTimers.man += dt * WALK_SPD.man;
if (walkTimers.man >= 1) {
  walkFrames.man = (walkFrames.man + 1) % 4; // 4 幀循環
  walkTimers.man -= 1;                         // 保留小數（不丟失精度）
}
💡 為何三個角色速度不同？

秒針每 60 秒繞一圈，分針每 60 分鐘，時針每 12 小時。小孩（秒針）轉最快，走路頻率設最高；男士（時針）轉最慢，走路緩慢沉穩，呼應「人生不同階段」的隱喻。

4.3Resize 事件與 Debounce

視窗大小改變時需要重新繪製背景，但 resize 事件在拖曳視窗時每秒觸發數百次。Debounce 技巧確保停止改變後才執行，避免效能浪費：

Debounce 防抖JS
let resizeBgTimer = null;

function resizeBgDebounced() {
  if (resizeBgTimer) clearTimeout(resizeBgTimer); // 取消上次的計時
  resizeBgTimer = setTimeout(resizeBg, 80);         // 80ms 後才執行
}

window.addEventListener('resize', resizeBgDebounced);
window.addEventListener('orientationchange', resizeBg);

// 手機 pinch zoom / 軟鍵盤彈出也會改變視口大小
if (window.visualViewport) {
  window.visualViewport.addEventListener('resize', resizeBgDebounced);
  window.visualViewport.addEventListener('scroll', resizeBgDebounced);
}

4.4時間數學 — 從時間到角度

將當前時間轉換成 [0, 1] 的比例值，再換算成弧度，是時鐘動畫的核心數學：

時間 → 分數 → 角度JS
const now = new Date();
const h  = now.getHours() % 12;  // 0–11
const m  = now.getMinutes();       // 0–59
const s  = now.getSeconds();       // 0–59
const ms = now.getMilliseconds(); // 0–999

// 時針：加入分、秒的微量，讓指針平滑移動
const hFrac = (h + m/60 + s/3600) / 12;  // [0, 1)

// 分針：加入秒的微量
const mFrac = (m + s/60 + ms/60000) / 60; // [0, 1)

// 秒針：包含毫秒，實現流暢的連續掃秒
const sFrac = (s + ms/1000) / 60;         // [0, 1)

// 換算成弧度（-π/2 使 0 對應 12 點方向）
const hA = hFrac * Math.PI * 2 - Math.PI/2;
const mA = mFrac * Math.PI * 2 - Math.PI/2;
const sA = sFrac * Math.PI * 2 - Math.PI/2;
🔢 為什麼要加次級單位？

如果時針只用小時數，每小時才跳動一次——不真實。加入 m/60（分鐘佔一小時的比例）後，時針會在一小時內平滑移動 1/12 圈，符合真實時鐘行為。秒針加 ms/1000 同理。

物件繪製 與移動

深入角色繪製系統：如何用 Canvas 基本圖形拼出有個性的角色， 以及如何用貝塞爾曲線繪製有機形狀，並讓角色「在軌道上走路」。

5.1角色結構：由內到外分層繪製

角色是由多個基本圖形（圓形、橢圓、線段、貝塞爾曲線）組合而成。繪製順序至關重要：先畫底層（四肢），後畫上層（身體、頭部），利用自然遮蔽創造立體感。

互動示範 — 角色分層繪製過程
點擊步驟按鈕觀察繪製過程
  1. 四肢（腿 + 腳 + 手臂） — 最底層，會被身體遮住一部分，製造「腿從身體下方伸出」的視覺效果
  2. 身體（radialGradient 橢圓） — 用徑向漸層製造立體感，高光在左上角
  3. 衣物細節（領帶、蝴蝶結） — 身體之上
  4. 頭部（漸層圓形） — 頭部置於身體右上方，製造側臉行走的感覺
  5. 五官細節（眼睛、嘴巴、臉頰紅暈） — 最頂層

5.2走路動畫：4 幀循環

走路動畫使用4 幀循環：每幀定義不同的腿部角度和身體 bob（上下晃動量），形成自然的步伐節奏。

4 幀走路資料JS
function drawMan(frame) {
  // frame = 0, 1, 2, 3（循環）
  const bob  = [0,   1.5, 0,   1.5][frame%4]; // 身體上下晃動
  const legL = [0.45,0.15,0.45,0.75][frame%4]; // 左腿角度（弧度）
  const legR = [0.75,0.45,0.75,0.15][frame%4]; // 右腿角度（與左腿互相）
  const armL = [-0.6,-0.2,-0.6,-1.0][frame%4]; // 手臂（與腿反向擺動）
  const armR = [-1.0,-0.6,-1.0,-0.2][frame%4];

  // 腿部座標計算：從身體底部 (0, R+bob) 向角度方向延伸 14px
  limb(0, R+bob,
       Math.sin(legL)*14,
       R+bob+Math.cos(legL)*14);
}

5.5指針繪製：手繪感的秘訣

時鐘指針不只是一條線，還有發光效果（Shadow Blur）、陰影偏移和反向短尾，這些細節讓指針有質感和重量感：

帶光暈效果的指針JS
function drawHand(angle, length, width, color) {
  const ex = CX + Math.cos(angle) * length; // 指針尖端
  const ey = CY + Math.sin(angle) * length;
  const tx = CX - Math.cos(angle) * length * 0.15; // 反向短尾
  const ty = CY - Math.sin(angle) * length * 0.15;

  ctx.save();
  // 1. 偏移陰影（製造懸浮感）
  ctx.beginPath(); ctx.moveTo(tx+2,ty+2); ctx.lineTo(ex+2,ey+2);
  ctx.strokeStyle = 'rgba(0,0,0,0.12)'; ctx.lineWidth = width+1; ctx.stroke();

  // 2. 主指針 + 發光效果
  ctx.beginPath(); ctx.moveTo(tx,ty); ctx.lineTo(ex,ey);
  ctx.strokeStyle = color;
  ctx.shadowBlur = 8;               // 發光模糊半徑
  ctx.shadowColor = color + '88';  // 半透明光暈
  ctx.stroke();
  ctx.shadowBlur = 0;               // ⚠️ 畫完後重置，否則影響後續繪圖
  ctx.restore();
}

6.2進度弧線

時鐘內圈有三道弧線，隨時間比例增長，直覺展示「已走過多少」。繪製方式是 arc() 只畫一段而非整圓：

進度弧線繪製JS
// 三個軌道同時繪製進度弧
[[R_H, hFrac, COL_H.light],
 [R_M, mFrac, COL_M.light],
 [R_S, sFrac, COL_S.light]].forEach(([r, f, c]) => {
  ctx.save();
  ctx.beginPath();
  ctx.arc(
    CX, CY, r,
    -Math.PI/2,              // 起始角：12 點方向
    -Math.PI/2 + f*Math.PI*2 // 結束角：比例換算
  );
  ctx.strokeStyle = c;
  ctx.lineWidth = 3;
  ctx.globalAlpha = 0.22; // 半透明，低調輔助
  ctx.stroke();
  ctx.restore();
});

6.3visualViewport API — 手機友善的視口

手機上有兩種不同的「視口」概念，普通 resize 事件有時無法捕獲所有變化：

API觸發時機適用場景
window.innerWidth瀏覽器視窗縮放桌面瀏覽器
window.visualViewportpinch-zoom、軟鍵盤彈出手機必備

安全的視口尺寸計算JS
function resizeBg() {
  const vp  = window.visualViewport;
  const dpr = window.devicePixelRatio || 1;

  // visualViewport 已考慮 offset（pinch zoom 後的捲動偏移）
  const vw = vp ? Math.ceil(vp.width  + vp.offsetLeft) : window.innerWidth;
  const vh = vp ? Math.ceil(vp.height + vp.offsetTop)  : window.innerHeight;

  // 以螢幕尺寸為下限，避免縮小後背景出現空白
  const fw = Math.max(vw, screen.width  / dpr);
  const fh = Math.max(vh, screen.height / dpr);

  bgCanvas.width  = fw * dpr;
  bgCanvas.height = fh * dpr;
  bgCanvas.style.width  = fw + 'px';
  bgCanvas.style.height = fh + 'px';
  bctx.setTransform(dpr, 0, 0, dpr, 0, 0);

  drawBgPattern(); // 重新繪製
}
